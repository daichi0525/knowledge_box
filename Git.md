## Git

以下に、Gitでよく使われるコマンドの一覧とそれぞれの具体的な説明を示します。

### Git リポジトリ関連コマンド

- **git init**:
  - **用途**: 新しいGitリポジトリを初期化します。
  - **実行例**:
    ```sh
    git init
    ```
  - **結果**: 現在のディレクトリが新しいGitリポジトリとして初期化されます。`.git`ディレクトリが作成されます。

- **git clone**:
  - **用途**: リモートリポジトリをローカルにクローンします。
  - **実行例**:
    ```sh
    git clone <リポジトリURL>
    ```
  - **結果**: 指定されたリモートリポジトリがローカルに複製されます。

- **git remote add**:
  - **用途**: 新しいリモートリポジトリを追加します。
  - **実行例**:
    ```sh
    git remote add <リモート名> <リポジトリURL>
    ```
  - **結果**: 指定されたリモートリポジトリがリモートリポジトリのリストに追加されます。

### Git コミット関連コマンド

- **git add**:
  - **用途**: 変更をステージングエリアに追加します。
  - **実行例**:
    ```sh
    git add <ファイル名>
    git add .
    ```
  - **結果**: 指定されたファイル（またはすべての変更）が次のコミットに含まれるようステージングされます。

- **git commit**:
  - **用途**: ステージングエリアの変更をコミットします。
  - **実行例**:
    ```sh
    git commit -m "コミットメッセージ"
    ```
  - **結果**: ステージングエリアにある変更がローカルリポジトリにコミットされます。

- **git commit --amend**:
  - **用途**: 最後のコミットを修正します。
  - **実行例**:
    ```sh
    git commit --amend -m "修正されたコミットメッセージ"
    ```
  - **結果**: 最後のコミットメッセージや内容を修正します。

### Git ブランチ関連コマンド

- **git branch**:
  - **用途**: ブランチの一覧を表示したり、新しいブランチを作成したりします。
  - **実行例**:
    ```sh
    git branch
    git branch <ブランチ名>
    ```
  - **結果**: 現在のブランチの一覧が表示されるか、新しいブランチが作成されます。

- **git checkout**:
  - **用途**: ブランチを切り替えたり、特定のコミットにチェックアウトします。
  - **実行例**:
    ```sh
    git checkout <ブランチ名>
    git checkout -b <新しいブランチ名>
    ```
  - **結果**: 指定されたブランチに切り替わるか、新しいブランチが作成されてそのブランチに切り替わります。

- **git merge**:
  - **用途**: 他のブランチを現在のブランチにマージします。
  - **実行例**:
    ```sh
    git merge <ブランチ名>
    ```
  - **結果**: 指定されたブランチの変更が現在のブランチにマージされます。

- **git rebase**:
  - **用途**: 一連のコミットを他のブランチに適用します。
  - **実行例**:
    ```sh
    git rebase <ブランチ名>
    ```
  - **結果**: 現在のブランチのコミットが指定されたブランチの先頭に再適用されます。

### Git ステータスとログ関連コマンド

- **git status**:
  - **用途**: 作業ディレクトリの状態を表示します。
  - **実行例**:
    ```sh
    git status
    ```
  - **結果**: 変更されているファイルやステージングされているファイルの状態が表示されます。

- **git log**:
  - **用途**: コミット履歴を表示します。
  - **実行例**:
    ```sh
    git log
    git log --oneline
    ```
  - **結果**: リポジトリのコミット履歴が詳細または要約形式で表示されます。

- **git diff**:
  - **用途**: 作業ディレクトリやステージングエリアの変更を表示します。
  - **実行例**:
    ```sh
    git diff
    git diff --staged
    ```
  - **結果**: 変更された行が表示されます。`--staged`オプションを使うとステージングエリアの変更が表示されます。

### Git リモート関連コマンド

- **git fetch**:
  - **用途**: リモートリポジトリから最新の変更を取得します。
  - **実行例**:
    ```sh
    git fetch <リモート名>
    ```
  - **結果**: 指定されたリモートリポジトリの変更がローカルに取得されますが、作業ブランチには反映されません。

- **git pull**:
  - **用途**: リモートリポジトリから変更を取得して現在のブランチにマージします。
  - **実行例**:
    ```sh
    git pull <リモート名> <ブランチ名>
    ```
  - **結果**: 指定されたリモートブランチの変更がローカルブランチにマージされます。

- **git push**:
  - **用途**: ローカルの変更をリモートリポジトリに送信します。
  - **実行例**:
    ```sh
    git push <リモート名> <ブランチ名>
    ```
  - **結果**: 指定されたブランチの変更がリモートリポジトリに送信されます。

### Git その他のコマンド

- **git stash**:
  - **用途**: 作業中の変更を一時的に保存します。
  - **実行例**:
    ```sh
    git stash
    git stash pop
    ```
  - **結果**: 変更が一時的に保存され、作業ディレクトリがクリーンになります。`pop`コマンドで保存した変更を復元できます。

- **git tag**:
  - **用途**: 特定のコミットにタグを付けます。
  - **実行例**:
    ```sh
    git tag <タグ名>
    git tag -a <タグ名> -m "タグの説明"
    ```
  - **結果**: 指定されたコミットにタグが付けられます。

- **git reset**:
  - **用途**: 指定した状態にリポジトリをリセットします。
  - **実行例**:
    ```sh
    git reset --hard <コミットハッシュ>
    git reset --soft <コミットハッシュ>
    ```
  - **結果**: `--hard`オプションを使うと作業ディレクトリも含めてリセットされ、`--soft`オプションを使うとステージングエリアは維持されます。

以上がGitの主要なコマンドの一覧とその具体的な動作と結果の説明です。これらのコマンドを使いこなすことで、バージョン管理が効率的に行えます。
